VUE

Vue Framework - Javier Sánchez Carrizo

Historia

Vue es un Framework de JavaScript desarrollado en 2014 por Evan You, un ex-trabajador de Google.

Este framework nació de la idea de hacer que los programadores ahorren tiempo no escribiendo tanto código gracias a que incluye un conjunto de herramientas para poder desarrollar páginas web de forma más cómoda.

Características
  • Curva de aprendizaje:

    •     Es más sencilla con respecto a otros frameworks Front-End como React o Angular.

  • Completamente Modular:

    •     A lo que me reifero es a que este framework te ofrece lo básico y cualquier funcionalidad extra tendrás que implementarla.

  • Reactividad:

    •     Este framework cuenta con propiedades reactivas que permite actualizar cualquier variable sin necesidad de     hacerlo manualmente.

  • DOM Virtual:

    •     En el caso de querer cambiar un elemento de la vista, Vue creará una réplica del DOM para que cuando     realicemos los cambios se hagan de manera más óptima.

  • Mixins:

    •     Esta es la función y lógica de los componentes que se puede reutilizar en otros componentes web

  • Componentes:

    •     Los componentes de Vue cuentan con un ciclo de vida que se puede controlar.

  • Páginas SPA:

    •     En VUE se pueden desarrollar páginas SPA, estas son páginas ya cargadas cuando el usuario entra a la web, esto     se consigue con Vue Router

Ventajas
  • Es muy liviano, pesa unicamente 18 Kb.
  • Arquitectura basada en componentes: Esto nos permite dividir grandes boques de información en bloques más pequeños.
  • Compatible con muchas herramientas: Este framework es compatible con multiples herramientas con poca o ninguna configuración, además proporciona pruebas unitarias, bibliotecas de pruebas extremos a extremo, sistema de enrutamiento flexibles y fáciles de usar, renderizado del lado del servidor (SSR).
  • Es facil de utilizar y además su curva de aprendizaje es mucho mayor a cualquier otro framework de front.
Desventajas
  • Complejidad de la reactividad: El problema reside en que el sistema de reactividad solamente devuelve los trozos de datos activados por lo que puede haber errores durante la lectura de los datos y es requerido que dichos datos sean aplanados.
  • La barrera linguística: Inicialmente Vue.js fue adoptado por los chinos es por ello que muchos foros y canales de discusión estuviesen principalmente en chino, aunque hoy en día esto no sea así y haya un soporte para múltiples idiomas hay todavía algunos idiomas con menos soporte que los demás.
  • Riesgo de exceso de flexibilidad puede desembocar a la generación de código espagueti debido a que todos los miembros del equipo tienen diferente opinión sobre cómo hacer las cosas.
